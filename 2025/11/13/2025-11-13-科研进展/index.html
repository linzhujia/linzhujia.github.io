机器学习对接脚本使用说明、寻优算法解析及环境依赖说明

一、脚本功能与使用方法

该脚本是一个基于Optuna的自动优化脚本，核心功能是通过调整指定输入参数（如a0、e0、polar等），调用外部程序（epoch2d）执行计算，再根据程序输出的Ising能量相关指标进行多目标寻优，最终找到最优参数组合。

使用步骤：

  1. 外部程序与环境准备

    • 确保系统中已安装epoch2d程序，脚本中默认调用路径为 /data/wuyanxi/epoch/epoch2d/bin/epoch2d，需根据实际安装路径修改；
    • 由于脚本使用mpirun启动epoch2d，需提前安装MPI并行计算框架（如OpenMPI），确保mpirun命令可正常执行。

  2. 数据库配置（可选，若需存储优化数据）

    • 脚本默认使用MySQL存储优化过程中的试验数据（参数、目标值等），需提前完成以下配置：
      1. 安装MySQL服务并启动（本地默认地址127.0.0.1）；
      2. 创建数据库（示例中为ion_acc），可通过MySQL命令行执行：CREATE DATABASE ion_acc;；
      3. 确保数据库用户（示例中为wuyanxi）拥有该数据库的读写权限，且密码正确（示例中为8311Yanxi）；
    • 若无需数据库存储，可修改optuna.create_study中的storage参数为storage=None，关闭数据库存储功能。

  3. 参数与目标函数调整

    • 待优化参数范围：在目标函数objective_func中，通过trial.suggest_*方法定义参数范围（如e0的trial.suggest_uniform('e0', 0, 1)表示在0~1之间均匀采样），可根据实际需求修改参数名称、类型（均匀分布、离散分布等）和范围；
    • 外部程序调用命令：修改subprocess.run中的命令字符串，将mpirun -np 40 /data/wuyanxi/epoch/epoch2d/bin/epoch2d替换为实际的epoch2d执行命令（如调整-np后的并行进程数）；
    • 目标函数：脚本默认优化两个目标（最小化energy_abs_mean、最大化energy_std），可在optuna.create_study的directions参数中修改（如['minimize']改为单目标优化）。

二、调用的寻优算法

脚本核心使用的是Optuna库中的TPESampler（Tree-structured Parzen Estimator Sampler），属于贝叶斯优化算法的经典实现。

算法细节：

  • 核心原理：TPESampler通过构建两个Parzen估计器（概率模型），分别拟合“优秀样本”（目标函数值满足要求的参数）和“普通样本”的参数分布，后续采样时优先选择使“优秀样本分布概率/普通样本分布概率”更大的参数，从而高效聚焦最优参数区域；
  • 初始化策略：脚本中n_startup_trials=40表示前40次试验使用随机搜索初始化，避免初始样本过少导致概率模型拟合偏差，40次后自动切换为TPESampler进行贝叶斯优化；

三、可替换的寻优算法

Optuna支持多种主流寻优算法，可通过修改sampler参数直接替换，适配不同场景（如单目标/多目标、连续/离散参数、高维/低维）。以下是常用替代算法及使用示例：

1. 随机搜索（RandomSampler）

  • 适用场景：参数空间未知、初步探索、低维参数优化；
  • 特点：无先验假设，随机采样参数，简单稳定，适合作为基准对比；

2. CMA-ES（协方差矩阵自适应进化策略）

  • 特点：基于进化策略，通过自适应调整协方差矩阵优化采样分布，鲁棒性强，收敛速度快；

3. 网格搜索（GridSampler）

  • 适用场景：参数维度少（≤3）、范围小、离散参数优化；
  • 特点：穷举所有参数组合，结果确定，无随机因素，但维度高时计算量爆炸；

四、所需安装的包及依赖

脚本运行需依赖Python库和系统工具，以下是详细安装说明（含版本要求、依赖关系）：

1. 必装Python库（核心依赖）

库名称 功能说明 安装命令 依赖说明
numpy 数值计算（数组处理、统计分析） pip install numpy>=1.21 无强制依赖，建议≥1.21以兼容其他库
optuna 优化框架（核心，提供所有采样器） pip install optuna>=3.0 自动依赖：scipy、sqlalchemy、tqdm、colorlog
matplotlib 静态绘图（保存能量谱图） pip install matplotlib>=3.5 依赖：numpy、pillow
plotly 交互式可视化（优化结果图表） pip install plotly>=5.0 依赖：tenacity、pandas（可选）
pymysql MySQL数据库连接（存储优化数据） pip install pymysql>=1.0 无额外依赖
sdf_helper 解析SDF格式文件（epoch2d输出格式） pip install sdf-helper 依赖：numpy、h5py（SDF文件底层依赖）

3. 安装注意事项与依赖冲突解决

（1）版本兼容性

  • optuna需≥3.0版本才能支持NSGAIISampler和多目标优化的完整功能，若安装旧版本会导致报错；
  • numpy版本建议统一为≥1.21，避免matplotlib、sdf_helper等库因numpy版本过低无法运行。

（3）特殊库安装问题

  • sdf_helper安装失败：若pip install sdf-helper无法找到包，可能是自定义库，需联系epoch2d提供者获取源码，通过pip install .本地安装；

总结

该脚本基于Optuna的TPESampler实现多目标参数优化，核心是“参数采样→外部程序调用→结果反馈→迭代优化”。运行前需安装Python库和MPI系统工具，可通过替换Optuna采样器适配不同优化场景（如随机搜索用于初步探索、NSGA-II用于多目标优化）。建议使用虚拟环境隔离依赖，避免版本冲突。
